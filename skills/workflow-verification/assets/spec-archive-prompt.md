# Spec Archive Subagent Prompt Template

Use this template when dispatching a spec archive subagent via the Agent tool. Invoked by `workflow-verification` when the user confirms archiving after all verification checks pass.

**Purpose:** Archive a completed feature spec directory to `docs/spec/archive/` and update the global Spec index — in a single atomic commit, executed by an independent subagent.

```
Task tool (general-purpose):
  description: "Archive feature_<date>_<id>_<topic> to docs/spec/archive/"
  prompt: |
    You are archiving a completed feature spec. Your job is precise and auditable.

    ## Source Directory

    docs/spec/feature_<date>_<id>_<topic>/

    Contains:
    - spec-design.md — design document
    - plan.md (or plan-<N>.md) — implementation plan with task checklist
    - verify-report.md — verification report (generated by /sdd-verify)
    - Other supporting files (diagrams, assets, etc.)

    ## Step 1: Move Directory to Archive

    ```bash
    # Ensure archive dir exists
    mkdir -p docs/spec/archive

    # Move the feature directory
    git mv docs/spec/feature_<date>_<id>_<topic>/ docs/spec/archive/feature_<date>_<id>_<topic>/
    ```

    ## Step 2: Update Global Spec Index

    Read `docs/spec/global/index.md`. You MUST update three things:

    ### 2.1 Add archive row to `## Archived Features` table

    Add one row at the end of the table (before any horizontal rule or next section):

    ```
    | [feature_<date>_<id>_<topic>](../archive/feature_<date>_<id>_<topic>/) | <one-line summary> | <domain> | <YYYY-MM-DD> |
    ```

    - **Summary**: Extract from `spec-design.md` title/overview, or `plan.md` "Goals" field. One sentence max.
    - **Domain**: Infer from the feature ID topic segment (e.g. `tool-search` → tool-search, `multi-agent` → agent). If the domain has its own page at `./domains/<domain>.md`, use the domain slug from that file. Common domains: agent, tui, mcp, storage, relay-server, langfuse, etc.
    - **Date**: Today's date in ISO format.

    ### 2.2 Update domain count in `## Domain Index`

    Find the row matching the domain in the bullet list. Increment its feature count by 1.

    If no matching domain row exists, add a new one:
    ```
    - [<domain>](./domains/<domain>.md) — <description> — N features
    ```
    Set N=1 for a new domain.

    ### 2.3 Update footer timestamp

    Find the `*Last updated:*` line at the bottom. Replace it:
    ```
    *Last updated: YYYY-MM-DD — updated when archiving feature_<date>_<id>_<topic>*
    ```
    If the line doesn't exist, add it after a `---` separator at the end of the file.

    ## Step 3: Commit

    ```bash
    git add docs/spec/archive/ docs/spec/global/index.md
    git commit -m "chore: archive feature_<date>_<id>_<topic> — all verifications passed"
    ```

    ## Verification

    After committing, confirm:
    - [ ] Feature directory no longer exists at `docs/spec/feature_<date>_<id>_<topic>/`
    - [ ] Feature directory exists at `docs/spec/archive/feature_<date>_<id>_<topic>/`
    - [ ] `docs/spec/global/index.md` has the new row in `## Archived Features`
    - [ ] `docs/spec/global/index.md` domain count is updated
    - [ ] `docs/spec/global/index.md` footer timestamp is current

    Report back exactly what was done: commit hash, index changes made, and the final archive path.
```
